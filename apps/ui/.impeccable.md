## Design Context

### Product

**aeqi** is an unopinionated agent runtime. Autonomous companies staffed by AI
agents. Four primitives: Agents (WHO), Events (WHEN), Quests (WHAT), Ideas
(HOW). The dashboard is the control surface.

### Users

Developers, founders, and technical operators managing autonomous AI agent
systems. They're in flow state — issuing commands, reading agent output,
iterating fast. The chat input and the sidebar are their primary control
surfaces. Every millisecond of friction costs focus.

### Brand personality

**Precise. Authoritative. Beautiful.**

Japanese-German fusion — the restraint of wabi-sabi meets Bauhaus
functionalism. Nothing decorative, nothing wasted. Every element earns its
place through utility. Beauty emerges from perfect proportion and purposeful
reduction. Institutional, calm, intelligent — Bloomberg × Palantir ×
protocol-layer register.

### Aesthetic direction — v5.1 "Graphite + Ink"

- **Four-tier elevation ladder** (settled 2026-04-27):
  - `--color-paper` (`#f3f3f4`) — outer shell, page bg, sidebar.
  - `--color-card` (`#fcfcfd`) — primary reading surface (whisper-tinted
    off-white, **not** flat white). Where most page content sits.
  - `--color-card-elevated` (`#ffffff`) — pure white. Surfaces that lift
    above the card: popovers, secondary buttons, cards-on-cards, the nav.
  - `--color-card-subtle` / `--color-card-muted` — inset family inside
    cards (gentle grouping / stronger trays).
- **Near-black ink at alpha** + **one near-black graphite accent**
  (`#0a0a0b`). Zero chroma on every surface and ink. Depth comes from
  value contrast across the ladder, not from drop shadows.
- **Ratio rule — hold or it goes generic:** ~70% paper/card · ~28% ink ·
  **~2% graphite accent**. The accent earns its weight by scarcity:
  wordmark, primary CTA, active nav, focus ring, selection. Trust through
  restraint, not through a brand colour.
- **Jade** (`#2e8f71`) is reserved for the **success** semantic so brand and
  status signal read distinct. Info now inherits the accent graphite.
- **Three typefaces, non-overlapping roles:**
  - **Inter** (`--font-sans`) — body, UI chrome, labels, IDs, code,
    model names, keybindings (everything that isn't a page title or
    the wordmark). Mono was purged — Inter does it all.
  - **Exo 2** (`--font-display`) — page titles, hero, marquee moments (geometric sci-fi sans, 500/600)
  - **Zen Dots** (`--font-brand`) — the **æqi** wordmark only
- **4px spacing grid.** Mathematical rhythm throughout.
- **32px row rhythm** (`--input-h` / `--sidebar-row-h`) — every dense
  interactive surface shares one height.
- **Motion conveys continuity, not delight.** `--transition-fast` 150ms,
  `--transition-normal` 200ms. No bouncy easing.
- **Shadows are restrained.** `--shadow-sm/md/lg` exist, but depth usually
  comes from surface value contrast (paper → card → elevated) rather than
  drop shadows.

### Anti-references

- Gradient backgrounds, glassmorphism as decoration, neon accents
- Rounded bubbly interfaces (Notion, Linear's softer elements)
- Anything that feels like "AI wrapper startup aesthetic"
- Dark mode with glowing accents
- Full-bleed color washes, hero gradient metrics
- Rounded rectangles with generic drop shadows
- Heraldic gold, warm parchment, royal blue, saturated cyan, **steel-blue
  accent + hue-tinted paper (v3)** — retired palettes, do not reintroduce
- Capitalized "AEQI" — always lowercase **æqi**

### Design principles

1. **Density over decoration** — Professionals scan, they don't browse. Every pixel communicates.
2. **Black ink on neutral paper** — The palette is restrained by conviction, not limitation. One accent earns its weight by scarcity.
3. **The input is the interface** — The composer is the most important element. It should feel like a precision instrument, not a text box.
4. **Quiet confidence** — No element begs for attention. Authority comes from restraint and perfect alignment.
5. **Japanese negative space, German grid** — Breathing room is intentional and mathematically placed, never arbitrary.
6. **Reach for the primitive first** — A new ad-hoc class is the last resort. The 25 primitives in `components/ui/` cover the common needs.

### Critique & polish discipline

Before tightening any product surface, run a 4-dimension check (the app
skips "innovation" — internal surfaces optimize for clarity, not
moments):

- **Hierarchy** — Is the primary action visually primary? Is one
  decorative element stealing attention from the work?
- **Execution** — Tokens, alignment, weight pairings. Tells: hand-rolled
  shadows, ad-hoc colors, off-grid spacing, two sizes used for one role.
- **Functionality** — Does every control do what its visual weight
  promises? Is anything decorative pretending to be interactive?
- **System fit** — Is this consuming primitives, or quietly forking? If
  forking, why — and does the forked behaviour belong upstream?

Then apply two editing rules:

- **120 / 80.** One detail per page gets the craft attention — usually
  the input, the toolbar, or the active row. Everything else stays at a
  calm 80%. Two competing 120%-treatments on the same screen flattens the
  hierarchy and sends the user nowhere.
- **Prune test.** For every element, ask: "if I removed this, would the
  user notice?" If no — remove it. Dead pixels are dead weight.

## Sources of truth (canonical artifacts)

Values drift when they live in prose. These files are the system — this doc
only describes how to use them:

| Artifact                                       | Role                                                                          |
| ---------------------------------------------- | ----------------------------------------------------------------------------- |
| `../../packages/tokens/src/tokens.css`         | **Values** — colors, fonts, spacing, radii, shadows, motion. The one source.  |
| `../../packages/tokens/src/tokens.ts`          | TypeScript mirror of tokens.css — for canvas / chart / JS consumers.          |
| `src/styles/primitives.css`                    | Dashboard-specific aliases and tokens (`--input-h`, `--sidebar-row-h`, etc).  |
| `src/components/ui/`                           | The primitives with `.module.css` + Storybook stories — the component API.    |
| `src/components/ui/Welcome.mdx`                | Storybook entry — sidebar map, where to look for what.                        |
| `src/components/ui/docs/Foundations*.mdx`      | **Foundations** — color, typography, spacing, radii, motion, principles.      |
| `src/components/ui/docs/Patterns*.mdx`         | **Recipes** — composed patterns. `PatternsToolbar.mdx` is the canonical       |
|                                                | search/sort/filter/view/+ New layout, including the chrome/paper/ink rule.    |
| [aeqi.ai/brand](https://aeqi.ai/brand)         | Public brand reference — palette, typography, identity, downloads.            |

When designing a feature or normalizing a page:

1. Read `Foundations/Principles` for the ratio rule and the four primitives.
2. Read `Patterns/Toolbar` for the canonical primitive-page header + the
   chrome (search/filter/view) → paper (Cancel/Back) → ink (Save/+ New)
   tier rule + the labeled-Button variant decision tree.
3. Read the matching `*.module.css` + `*.stories.tsx` to see the API surface.
4. Use tokens for every color / font / spacing / radius / transition — never
   hardcode. If a token is missing, add it to `tokens.css` first.
5. If a need doesn't map to any existing primitive, propose a new primitive
   before writing a page-level class.

## Locked rules (also rendered in Storybook for the live recipe)

These are non-negotiable and survive across sessions:

- **Toolbar = two zones.** Chrome left+middle (search, sort/filter/view, scope-as-filter)
  with `--input-bg` muted-tint background. Action right (Cancel, Save, +New, Delete)
  with `--color-card` (white) for `secondary` or `--color-accent` (ink) for `primary`/`danger`.
  See `Patterns/Toolbar`.
- **`<Button>` variants — affordance identity wins.** The same action gets the same
  variant on every surface. Two primaries in one toolbar are fine when targeting
  distinct outcomes (e.g. `+ New` and `Save`). Navigation (Back, breadcrumb) is
  always `secondary`. `ghost` is the quiet inline action whose hover reveals it —
  never the Cancel sibling beside a primary.
- **White / black / grey.** Resting `secondary` buttons paint white (paper),
  not grey (chrome). `--btn-secondary-bg = --color-card`. Grey is reserved
  for chrome (page background, dividers, quiet typography, display-modifier
  buttons). `--color-bg-base` is page chrome — never use it as a button
  background.
- **Lowercase is brand, not labels.** `aeqi` (and `æqi`) is the only string
  that stays lowercase in product copy. Scope / lifecycle / status nouns
  (Self, Global, Runtime, Webhooks, Routines, …) are Title Case at the
  display layer. Identifiers (URL params, JSON keys) keep their lowercase
  form.
