## Design Context

### Product

**aeqi** is an unopinionated agent runtime. Autonomous companies staffed by AI
agents. Four primitives: Agents (WHO), Events (WHEN), Quests (WHAT), Ideas
(HOW). The dashboard is the control surface.

### Users

Developers, founders, and technical operators managing autonomous AI agent
systems. They're in flow state — issuing commands, reading agent output,
iterating fast. The chat input and the sidebar are their primary control
surfaces. Every millisecond of friction costs focus.

### Brand personality

**Precise. Authoritative. Beautiful.**

Japanese-German fusion — the restraint of wabi-sabi meets Bauhaus
functionalism. Nothing decorative, nothing wasted. Every element earns its
place through utility. Beauty emerges from perfect proportion and purposeful
reduction. Institutional, calm, intelligent — Bloomberg × Palantir ×
protocol-layer register.

### Aesthetic direction — v4 "Graphite + Ink"

- **Pure neutral grey paper canvas** (`#f4f4f5`) + **flat white card**
  (`#ffffff`) + **near-black ink at alpha** + **one near-black graphite
  accent** (`#0a0a0b`). Zero chroma on every surface and ink.
- **Ratio rule — hold or it goes generic:** ~70% paper/card · ~28% ink ·
  **~2% graphite accent**. The accent earns its weight by scarcity:
  wordmark, primary CTA, active nav, focus ring, selection. Trust through
  restraint, not through a brand colour.
- **Jade** (`#2e8f71`) is reserved for the **success** semantic so brand and
  status signal read distinct. Info now inherits the accent graphite.
- **Four typefaces, non-overlapping roles:**
  - **Inter** (`--font-sans`) — body, UI chrome, labels
  - **JetBrains Mono** (`--font-mono`) — code, IDs, model names, keybindings
  - **Exo 2** (`--font-display`) — page titles, hero, marquee moments (geometric sci-fi sans, 500/600)
  - **Zen Dots** (`--font-brand`) — the **æqi** wordmark only
- **4px spacing grid.** Mathematical rhythm throughout.
- **32px row rhythm** (`--input-h` / `--sidebar-row-h`) — every dense
  interactive surface shares one height.
- **Motion conveys continuity, not delight.** `--transition-fast` 150ms,
  `--transition-normal` 200ms. No bouncy easing.
- **Shadows are restrained.** `--shadow-sm/md/lg` exist, but depth usually
  comes from surface value contrast (paper → card → elevated) rather than
  drop shadows.

### Anti-references

- Gradient backgrounds, glassmorphism as decoration, neon accents
- Rounded bubbly interfaces (Notion, Linear's softer elements)
- Anything that feels like "AI wrapper startup aesthetic"
- Dark mode with glowing accents
- Full-bleed color washes, hero gradient metrics
- Rounded rectangles with generic drop shadows
- Heraldic gold, warm parchment, royal blue, saturated cyan, **steel-blue
  accent + hue-tinted paper (v3)** — retired palettes, do not reintroduce
- Capitalized "AEQI" — always lowercase **æqi**

### Design principles

1. **Density over decoration** — Professionals scan, they don't browse. Every pixel communicates.
2. **Black ink on neutral paper** — The palette is restrained by conviction, not limitation. One accent earns its weight by scarcity.
3. **The input is the interface** — The composer is the most important element. It should feel like a precision instrument, not a text box.
4. **Quiet confidence** — No element begs for attention. Authority comes from restraint and perfect alignment.
5. **Japanese negative space, German grid** — Breathing room is intentional and mathematically placed, never arbitrary.
6. **Reach for the primitive first** — A new ad-hoc class is the last resort. The 18 primitives in `components/ui/` cover the common needs.

## Sources of truth (canonical artifacts)

Values drift when they live in prose. These files are the system — this doc
only describes how to use them:

| Artifact                               | Role                                                                          |
| -------------------------------------- | ----------------------------------------------------------------------------- |
| `../../packages/tokens/src/tokens.css` | **Values** — colors, fonts, spacing, radii, shadows, motion. The one source.  |
| `../../packages/tokens/src/tokens.ts`  | TypeScript mirror of tokens.css — for canvas / chart / JS consumers.          |
| `src/styles/primitives.css`            | Dashboard-specific aliases and tokens (`--input-h`, `--sidebar-row-h`, etc).  |
| `src/components/ui/`                   | The 18 primitives with `.module.css` + Storybook stories — the component API. |
| `src/components/ui/DesignLanguage.mdx` | **Rules**: how to use each primitive, when to use which, anti-patterns.       |
| [aeqi.ai/brand](https://aeqi.ai/brand) | Public brand reference — palette, typography, identity, downloads.            |

When designing a feature or normalizing a page:

1. Read `DesignLanguage.mdx` for the primitive-replaces-class map.
2. Read the matching `*.module.css` + `*.stories.tsx` to see the API surface.
3. Use tokens for every color / font / spacing / radius / transition — never
   hardcode. If a token is missing, add it to `tokens.css` first.
4. If a need doesn't map to any existing primitive, propose a new primitive
   before writing a page-level class.
