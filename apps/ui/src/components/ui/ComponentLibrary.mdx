# Component inventory

This page is the canonical inventory of primitives exported from
`@/components/ui`. The Welcome page tells you how the sidebar is organised;
this page tells you what is in the box.

For visual judgment, use **Get Started -> System Coherence**. It renders
the primitives together as an app surface so radius, motion, density, and
surface tiers can be reviewed as one system.

Every primitive in the library satisfies three conditions:

- it is exported from `src/components/ui/index.ts`
- it has at least one Storybook story
- it consumes shared tokens instead of ad hoc local styling

## Primitives by bucket

The sidebar mirrors these buckets. If you add a new primitive, retitle its
story `Primitives/<bucket>/<name>` to keep the navigation aligned.

### Actions

| Primitive     | Purpose                                                |
| ------------- | ------------------------------------------------------ |
| `Button`      | Primary, secondary, ghost, and destructive actions     |
| `IconButton`  | Dense icon-only actions for toolbars and rows          |
| `CardTrigger` | Keyboard-safe clickable cards and choice rows          |
| `TabTrigger`  | Standalone tab-like trigger for dense mode controls    |
| `ChipClose`   | Small remove affordance for chips and selected filters |
| `Menu`        | Disclosed action list for overflow and row controls    |

### Inputs

| Primitive      | Purpose                                                    |
| -------------- | ---------------------------------------------------------- |
| `Input`        | Single-line text input with label, hint, and error support |
| `Textarea`     | Multi-line prose input using the same field language       |
| `Select`       | Native single-choice dropdown with consistent chrome       |
| `SelectOption` | Option row for custom listbox/select compositions          |
| `Combobox`     | Searchable single-choice picker over a known option set    |

### Containers And Layout

| Primitive | Purpose                                               |
| --------- | ----------------------------------------------------- |
| `Card`    | Generic grouped container with optional header/footer |
| `Tabs`    | Accessible tablist with counts and panel content      |
| `Stack`   | Tokenized vertical spacing                            |
| `Inline`  | Tokenized horizontal spacing                          |

### Data Display

| Primitive     | Purpose                                           |
| ------------- | ------------------------------------------------- |
| `Badge`       | Status and metadata chips                         |
| `StatusRow`   | Row-level status with semantic dot and label      |
| `DetailField` | Label/value pair for settings and detail views    |
| `Icon`        | Lucide-backed icon wrapper with system sizing     |
| `QRCode`      | Local-only QR rendering for sensitive setup flows |
| `Table`       | Canonical tabular data layout                     |
| `TagList`     | Tag/chip list with optional empty fallback        |

### Overlays

| Primitive       | Purpose                                           |
| --------------- | ------------------------------------------------- |
| `Modal`         | Dialog shell for confirmations and forms          |
| `ConfirmDialog` | Canonical confirmation flow on top of Modal       |
| `Popover`       | Anchored floating surface for filters and pickers |
| `Tooltip`       | Hover/focus explanation for dense controls        |

### Feedback

| Primitive       | Purpose                                         |
| --------------- | ----------------------------------------------- |
| `Loading`       | aeqi wordmark loading indicator                 |
| `ThinkingDot`   | Agent-thinking indicator for streaming surfaces |
| `EmptyState`    | No-data and zero-result placeholders            |
| `Banner`        | Inline feedback for notices and warnings        |
| `ErrorBoundary` | Render fallback UI when a subtree throws        |

## Adding a new primitive

1. Start from an existing one (`Input.tsx` / `Input.module.css` are the cleanest template).
2. Props: `variant`, `size`, `className`, then role-specific props. Always accept `className` and forward it last so call sites can extend.
3. CSS module: use `--input-*`, `--space-*`, `--radius-*`, `--text-*` tokens only.
4. Add a Storybook story showing every variant and every size. Title it `Primitives/<bucket>/<name>`.
5. Export from `src/components/ui/index.ts`.
6. Add the row to the bucket table above.

The `design-system:audit` script fails when an exported primitive has no
Storybook story. If a primitive lands in code but not in this inventory, the
library docs are incomplete.
