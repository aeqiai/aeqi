# aeqi component library

The reference for every primitive exported from `@/components/ui` and
the rules that keep them coherent. Use it to look up a component, copy
a working example, inspect a token, or see how primitives compose into
product patterns.

## Find what you need

| Goal                                      | Where to look                          |
| ----------------------------------------- | -------------------------------------- |
| Browse every primitive grouped by purpose | **Primitives** in the sidebar          |
| Inspect a design token (click to copy)    | **Foundations → Color** (and siblings) |
| See the design rules and philosophy       | **Foundations → Principles**           |
| See composed UI recipes                   | **Patterns** in the sidebar            |
| See the import surface and inventory      | **Get Started → Component Library**    |
| Jump to a specific primitive by name      | Press <kbd>/</kbd> to search           |

## How the sidebar is organised

### Get Started

Orientation pages. Read these once. Come back to the Component Library
when you need the import surface or the "which primitive when" table.

### Foundations

The design system itself — one scrollable page per concern so each is a
single read:

| Page           | What's in it                                         |
| -------------- | ---------------------------------------------------- |
| **Principles** | Ratios, restraint rules, the four primitives         |
| **Color**      | Surfaces, ink ladder, accent, status — click to copy |
| **Typography** | Four font roles with live specimens + type scale     |
| **Spacing**    | 4px grid visualised, row rhythm rules                |
| **Radii**      | Six-step scale with live shapes                      |
| **Elevation**  | Canonical lifted card vs alternatives                |
| **Motion**     | Three cadences you can play side-by-side             |
| **Wordmark**   | The æqi brand mark and its rules                     |

### Primitives

The 25 primitives, grouped by purpose so you don't scan a flat list:

| Bucket           | What lives here                                                          |
| ---------------- | ------------------------------------------------------------------------ |
| **Actions**      | `Button`, `CardTrigger`, `ChipClose`, `IconButton`, `Menu`, `TabTrigger` |
| **Inputs**       | `Input`, `Textarea`, `Select`, `SelectOption`, `Combobox`                |
| **Containers**   | `Accordion`, `Card`, `Tabs`                                              |
| **Data Display** | `Badge`, `DetailField`, `Icon`, `TagList`                                |
| **Overlays**     | `Modal`, `Popover`, `Tooltip`                                            |
| **Feedback**     | `Spinner`, `ThinkingDot`, `EmptyState`, `ErrorBoundary`                  |
| **Layout**       | `Stack`, `Inline`                                                        |

Every primitive carries `autodocs`, so each entry has a generated
**Docs** tab with prop tables alongside the hand-written stories.

### Patterns

Composed recipes that show how primitives combine into dashboard UI.
Use these as starting templates, not as fixed screenshots.

| Pattern             | What it composes                                                                                                             |
| ------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| **Toolbar**         | Search + sort/filter/view + Button — the canonical primitive-page header, chrome/paper/ink tier rules, variant decision tree |
| **Layout**          | Stack + Inline — the canonical replacements for ad-hoc flex                                                                  |
| **Agent Card**      | Card + Badge + DetailField + TagList + Button                                                                                |
| **Quest Row**       | Badge + IconButton (dense list rhythm)                                                                                       |
| **Empty Dashboard** | EmptyState + Button (first-run, zero-data)                                                                                   |

- <kbd>D</kbd> — switch to the docs view
- <kbd>C</kbd> — switch to the canvas view

## Conventions

- Stories live next to their primitive in `apps/ui/src/components/ui/`.
- Story title format is `Primitives/<bucket>/<primitive>` — match this
  when you add a new one.
- Foundations and Patterns pages live in `apps/ui/src/components/ui/docs/`
  so the primitive folder stays scannable.
- Use real strings from the dashboard ("Create Quest", "Assign Agent")
  in examples, not "Lorem ipsum".
- Never hardcode hex, radius, or transition values in a story — reach
  for a token. The **Foundations → Color / Radii / Motion** pages are
  the source of truth.

If a primitive ships in code without a story, the library is incomplete.
