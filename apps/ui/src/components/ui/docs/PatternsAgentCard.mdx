# Agent Card

Compact summary of an agent on the dashboard or in a grid view.
Composes `Card`, `Badge`, `DetailField`, `TagList`, `IconButton`, and
`Button`.

## Anatomy

1. **Header row** — mono slug + human name on the left, status `Badge`
   on the right.
2. **Detail ladder** — three `DetailField` rows at most. More than three
   and the card stops being a summary.
3. **Tags** — `TagList` for expertise signals. Keep to 3–5 tags.
4. **Action bar** — up to two actions, separated from the body by a
   `--space-4` gap and a hairline divider.

## When to use

Dashboard grids. Agent pickers. Anywhere the user is scanning a set of
agents and needs to decide which one to open.

## When not to use

- For a single, focused agent view → use `Panel` with a full detail
  layout instead.
- For a list with >20 agents → rows (see **Patterns → Quest Row**) scan
  faster than cards.
