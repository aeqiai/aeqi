# Quest Row

Dense list row for a quest queue. Reads left-to-right: status, title,
agent, timestamp, actions.

## Anatomy

1. **Status badge** — fixed-width column. Columns line up across the whole
   list, so statuses scan vertically.
2. **Title** — single line, truncated on overflow. Never wrap to two
   lines in a list row.
3. **Agent** — mono so it reads as an identifier, not prose.
4. **Timestamp** — tabular-nums so digits line up; right-align if the
   column is narrow.
5. **Row actions** — one `IconButton` with an overflow menu, not an
   inline action bar. Row actions are secondary; the row itself is the
   primary action (click to open).

## When to use

Quest queues, event streams, agent activity lists — anywhere the user is
scanning to find one item among many.

## When not to use

- For 3–8 items with more than 2 data points each → use a **Card grid**
  (see Agent Card) so each item has room to breathe.
- For tabular data that needs sortable columns and filters → reach for a
  real `<table>` with column headers, not a row pattern.
