# Breakpoints

aeqi is designed desktop-first. Breakpoints exist for page layouts only — the
primitives themselves use flex and grid to stay fluid. The token system has
five breakpoints, but `lg` is home base; the others earn their use.

## The breakpoints

| Token    | Width  | Use case                                    |
| -------- | ------ | ------------------------------------------- |
| --bp-sm  | 640px  | Hand-held, one-hand reach (landscape phone) |
| --bp-md  | 768px  | Small tablet, split-pane laptop             |
| --bp-lg  | 1024px | Laptop default; the design's home base      |
| --bp-xl  | 1280px | Wide desktop, multi-rail dashboards         |
| --bp-2xl | 1536px | Ultra-wide; rarely needed                   |

## Usage

### Page layout CSS

Write media queries with the actual pixel value (CSS variables cannot be used
inside media query definitions in all browsers):

```css
@media (min-width: 768px) {
  .split-pane {
    display: grid;
    grid-template-columns: 1fr 1fr;
  }
}
```

The CSS token is the source of truth. If you change `--bp-md` in `tokens.css`,
the breakpoint updates everywhere it's used.

### JavaScript layouts

For dynamic layouts, matchMedia, or window width checks, import from `@aeqi/tokens`:

```ts
import { breakpoint } from "@aeqi/tokens";

if (window.innerWidth >= parseInt(breakpoint.lg)) {
  // show multi-column layout
}
```

## Don't

- **Don't add a sixth breakpoint** without changing the design system. Five is
  the contract. If a layout needs a unique threshold, that's a sign the layout
  is wrong, not that the system needs another breakpoint.
- **Don't use breakpoints inside primitives.** Primitives must work at any
  container width. Breakpoints belong on page layout layers only.
- **Don't pick px values ad-hoc in component CSS.** If a layout can't fit the
  five breakpoints, work with design to adjust the layout, not to add a new
  threshold.
