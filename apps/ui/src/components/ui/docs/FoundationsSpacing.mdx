# Spacing

A single 4px-grid scale. Pick the closest step; do not introduce off-grid
values.

<SpacingScale />

## Row rhythm

The app uses one canonical row height, `--input-h` (32px), for every
dense interactive surface: sidebar nav rows, inputs, buttons at `md`,
scope indicators. Buttons and inputs placed on the same row line up
without tweaks.

The composer is taller (multi-line) but borrows the same border, radius,
and colors so it reads as a member of the same family.

- Use `sm` size to feel denser.
- Use `lg` size for hero and auth surfaces.
- Do not introduce new row heights.

## Do / don't

- **Do** nest spacing tokens the same way you nest layout. Panel padding
  at `--space-6`, inner form gaps at `--space-4`, inline gaps at
  `--space-2`.
- **Do** reach for `gap` + `flex`/`grid` before margin stacks.
- **Don't** use pixel values directly. If the step you need isn't in the
  scale, the design is fighting the system.
