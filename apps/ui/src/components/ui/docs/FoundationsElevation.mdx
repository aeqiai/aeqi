# Elevation

Two ways to lift a surface above the page:

1. **Value contrast across the v5.1 ladder** (paper → card → elevated).
   This is the default. A card sitting on paper is already lifted by
   the tonal step; an elevated surface sitting on a card lifts again
   to pure white. Depth without shadows.
2. **The `--card-elevation` token** for surfaces that need a hairline
   ring + whisper of ambient shadow (e.g. a working sheet that floats
   over a tinted frame). Composes a 1px inset ring (via `box-shadow`,
   no layout cost) with restrained ambient + long-range soft shadows.

The sheet separates without any of the drop-shadow heaviness that reads
as "AI wrapper."

## Usage

```css
.sheet {
  background: var(--color-card);
  border-radius: var(--radius-lg);
  box-shadow: var(--card-elevation);
}
```

- Pair `--card-elevation` with `--radius-lg` and `--color-card`.
- Do **not** combine with `--shadow-md`, `--shadow-lg`, or custom drop
  shadows. The token already carries its own depth.
- The inline `--shadow-popover` / `--shadow-glow` tokens are for floating
  surfaces (popovers, pickers). Lifted cards use `--card-elevation`.

## Do / don't

- **Do** use elevation to separate a working sheet from its tinted frame.
- **Do** keep the hairline ring; it reads crisp on both light and dark
  backgrounds.
- **Don't** stack two elevation tokens on the same surface. If the card
  needs more weight, reach for contrast, not for bigger shadows.
