- `/home/claudedev/aeqi-platform/src/server.rs` line 447 registers the route in `auth_billing` router.
- Commit timestamp: 07:12 CEST.

Live binary audit:
- `aeqi-platform.service` started at 03:55 CEST (binary build time: same).
- Binary predates commit by ~3.3 hours — new stub not compiled in.

Observed behavior:
- `POST https://app.aeqi.ai/api/wallet/upgrade-to-passkey` with auth, no X-Entity header → **HTTP 400** `{"error":"X-Entity header required"}`.
- This happens because the route falls through to the catch-all proxy (`/api/{*rest}` in `proxied` router), which calls `extract_entity_id()`, which requires X-Entity.
- The correct 501 path is in `auth_billing` router and would match BEFORE the catch-all — but only if the binary contains the new registration.
- Impact: frontend `WalletUpgradeSection.tsx` checks `if (msg.includes("501") || msg.toLowerCase().includes("not implemented"))` — this check never fires on 400. User sees a red error banner on "Upgrade to passkey."

**Action required**: Deploy aeqi-platform after next Rust change (or standalone `./deploy/deploy.sh` in aeqi-platform).
