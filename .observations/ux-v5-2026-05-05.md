### Pill Buttons — P0 Assessment

The v4 report classified pill buttons as P1. The redeploy changed nothing here — 17/27 routes still have action pill buttons at 999px radius. The CSS bundle has exactly 12 `999px` occurrences (confirmed via live CSS download). After a confirmed successful redeploy, this is a real bug:

- The Wave 16 commit `94116b5b` targeted specific CSS classes but missed: auth buttons ("Continue", "Email me a code", "Google", "GitHub"), settings buttons ("Change", "Save"), list-action buttons ("New", "Import"), blueprint CTA ("New company"), "Resubscribe" button, governance role-label elements ("Director · founder" rendered as `<button>`).
- The `GovernancePage.tsx` hardcodes `borderRadius: "999px"` inline (not via CSS class) at three locations — not reachable by a CSS rule fix.
- Cookie consent buttons on landing pages are third-party (not in our CSS) — mark as won't-fix.

**Reclassify to P0.** A fix that ships but doesn't land on the buttons users interact with is a broken fix. The correct resolution is a global `.btn { border-radius: var(--radius-sm) }` rule that cascades through all surfaces, plus inline style removal from GovernancePage.tsx.

### Plan Name UUID — P0 Assessment

The `d243bcf7` commit has NOT reached production through two consecutive deploys (v4 and v5). The PLAN section on company-settings still renders the raw entity UUID as both the primary identifier row and the h2-equivalent label. This is structural — the data binding fetches entity_id where plan name or plan tier label should appear. The commit may have targeted the wrong field or the wrong component. Needs investigation.
