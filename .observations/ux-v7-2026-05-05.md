## New Issues (v7 discoveries)

### N1: Director UUID occupant — wrong fix
`78c2532f` fixed the null case (`occupant_id ?? "Unoccupied"`) but the Director role has `occupant_id` set to the user UUID `bbbd909d...`. The UI renders the UUID because it doesn't resolve the occupant UUID to a display name. Fix: on the role card/list, join the occupant UUID against the entity members list and display the resolved name (or "Unoccupied" if the UUID doesn't resolve). This is a data-join in the frontend, not a null-coalescing.

### N2: me-treasury P3 downgrade
Personal entity treasury copy says "This Company isn't billed through Stripe yet" — should say "This account..." or "This personal entity..." for /me/* routes. The P3 from v6 is confirmed still present.

### N3: AEQI_UPPERCASE_STRUCTURAL session-rail FP still active
The structural AEQI detector still fires 7x on agent-overview/sessions because the session-rail ASIDE is in scope. `SESSION_RAIL_AEQI_INFO` v7 check confirms `AEQI_UPPERCASE_TOTAL=1x` — confirming these are all FPs. The detector needs to exclude session-rail elements from the structural scan.
