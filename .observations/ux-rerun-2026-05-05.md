## New issues

### N-1 — Personal rail sidebar shows full company rail (P2)

`/me/inbox`, `/me/agents`, etc. show the company sidebar sections (ORGANIZATION: Treasury · Ownership · Governance · Roles) in addition to WORKSPACE (Agents · Events · Quests · Ideas). Per `project_personal_rail_v1.md`, the personal entity rail should hide Roles/Ownership/Governance (degenerate — no external owners, no governance). The personal rail should be: Inbox · Agents · Events · Quests · Ideas · Treasury · Settings.

Current render shows the full company rail with ORGANIZATION header. This is not a blocker (the routes work) but violates the personal rail spec.

### N-2 — Quest titles contain "AEQI" uppercase (P3, data not UI)

`/me/quests` "Done" column shows repeated "AEQI DAO Pivot Implementation Plan" quest titles — 5+ duplicates visible. Two issues: (a) "AEQI" uppercase in user-visible data (brand rule: always lowercase "aeqi" in product copy); (b) duplicate quest creation — the same title appears 5 times, likely from a runaway agent loop. Data hygiene, not a routing regression. Screenshot: `me-quests.png`.

### N-3 — Ideas descriptions contain "AEQI" uppercase (P3, data not UI)

`/me/ideas` list shows stored ideas with "AEQI agent introduced its core capabilities", "The AEQI agent specializes in...", etc. This is agent-generated knowledge stored as ideas — the brand rule applies to UI copy, not agent-generated data. Low severity, no action on the UI layer.
