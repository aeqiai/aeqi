# How to manage yourself as an aeqi agent

An aeqi agent is not just a chat persona. It has durable ideas, quests,
events, roles, tools, and activity. Self-management means keeping those
primitives coherent enough that the next session can continue without
reconstructing the past from chat.

## Default loop

1. Search relevant ideas before non-trivial work.
2. Use a quest when the work should survive the current turn.
3. Store durable decisions, preferences, procedures, and evidence as ideas.
4. Update identity only when the agent's standing role or behavior changes.
5. Use events for context loading and agreed recurring routines.
6. Close or rewrite stale quests; do not let abandoned work define reality.
7. End with evidence: what changed, how it was checked, and what remains.

## What to store

- Stable user preferences.
- Operating rules the company should obey later.
- Reusable procedures for tools, deploys, reviews, or rituals.
- Decisions with rationale and consequence.
- Package choices and why they fit.
- Known failure modes and recovery steps.

## What to avoid

- Do not store raw chat just because it happened.
- Do not create events for wishes; create them for routines the Director
  wants to repeat.
- Do not hire an agent before a recurring responsibility is visible.
- Do not edit identity to capture one-off context.
- Do not leave quests open when they have no owner or evidence of done.

## Health check

Ask these questions during review:

- Can a new session understand the current priority from ideas alone?
- Are active quests still actionable?
- Do events load useful context without surprising the Director?
- Are roles and agent occupants still accurate?
- Did the agent learn anything durable that should be stored or superseded?
