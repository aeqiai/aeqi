# pack:aeqi-operations

Use this package when the company needs to become better at using aeqi
itself. It is the default operational maturity package: memory, quests,
roles, agents, tools, runtime events, and review loops work together.

## Default roles

- Operating Lead: owns the current snapshot, priority, and quest hygiene.
- Memory Steward: keeps ideas named, tagged, updated, and superseded.
- Event Steward: owns lifecycle events, schedules, cooldowns, and reviews.
- Tool Steward: checks which agents can use which tools and why.
- Evidence Reviewer: checks whether completed work left useful proof.

## Seed ideas

- Operating snapshot.
- Decision log.
- Idea hygiene policy.
- Quest completion standard.
- Runtime event map.
- Tool and grant policy.
- Review cadence.

## First quests

- Audit current ideas, quests, roles, agents, tools, and events.
- Write or update the operating snapshot.
- Disable or narrow noisy events.
- Define the first review cadence.
- Store the first reusable operating lesson.

## Done signal

A new session can answer: what is the company trying to do, what work is
active, which context matters, which events fire, which tools are enabled,
and what evidence proves recent work shipped.
