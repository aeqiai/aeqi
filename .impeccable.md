# aeqi — design context for impeccable

## Design Context

### Users

aeqi is the runtime for autonomous companies — humans direct, agents work. The home page (`/`) is the **director's bridge**: the place a founder lands multiple times a day to triage what their agents have surfaced for them. Users are technical, decisive, and impatient with marketing-shaped UI on a working tool. They expect dense interfaces, mono-precise IDs, and zero pixel waste.

The target reader is closer to a Linear power user or a Bloomberg terminal operator than a SaaS dashboard "first-time user." Onboarding matters but is not where the daily value is — daily value is in the inbox + agent surfaces, used heads-down.

### Brand personality

Three words: **decisive, restrained, hand-tooled.**

Reference register: Linear × Things × Vercel docs × ABC Dinamo type specimens. Trust through restraint, not through a brand colour. Hand-tooled means: every pixel intentional, no stock dashboard tropes, no "made by a designer" flourishes that don't earn their keep. The interface should read as a precision instrument the founder happens to also enjoy looking at.

Voice: lowercase prose. "aeqi" is **always** lowercase in copy (not in code identifiers — there `AEQI` casing is fine). Microcopy is direct, second-person, no apologies. "you're caught up" not "Great job, all done!".

### Aesthetic direction

**Design system v4 — Graphite + Ink.** Already deeply established; do not reinvent.

- **Surfaces** — pure-neutral greys, **zero chroma** on every grey. Four tiers in the ramp: `--color-card` (#fff, primary reading surface) → `--color-card-subtle` (#f8f8f9) → `--color-card-muted` (#ededf0) → `--color-paper` (#f4f4f5, outer shell). No warm-cream or cool-steel tints — that was retired with v3.
- **Ink** — `--color-ink` is `#0a0a0b` (near-black, never pure black). The text scale is alpha-stepped: 92% titles → 85% body → 54% secondary → 36% eyebrow → 22% disabled.
- **Accent** — `--color-accent: #0a0a0b` (the same near-black). Hover pushes to pure `#000`. Black IS the accent. Used at ~2% visual weight: primary CTAs, focus rings, active nav, selected items.
- **Jade `#2e8f71`** is **reserved for success only** — completed quests, resolved events, "shipped" states. Never use it for "active" / "primary" / "highlight". Status signal must read visually distinct from brand.
- **Theme**: light. The whole product runs on paper + card surfaces with the tinted shell. Dark mode is not on the roadmap.

**Typography signature** — already chosen, must be honored:
- `Inter` — body / UI text
- `JetBrains Mono` — IDs, counts, code, relative timestamps (tabular-nums always on)
- `Exo 2` — display, hero greetings, marquee headings (the "geometric sci-fi" gesture)
- `Zen Dots` — **only** the æqi wordmark. Never a heading, never copy. The dotted geometry is the brand mark and only the brand mark.

Modular type scale. Not fluid in app UI — fixed `rem` ladder per Linear/Things conventions. Fluid `clamp()` only on marketing-shaped pages (the founder's signup / blueprints surface, not the dashboard).

### Anti-references — visual patterns to NEVER use

- **Cards-of-stats hero with sparklines.** Generic AI dashboard tell.
- **Border-left coloured stripes** on rows / cards / alerts. Already on impeccable's absolute-ban list; AEQI memory has rejected this twice independently.
- **Glassmorphism / glow / neon-on-dark.** Anti-AEQI.
- **Gradient text** — banned outright (impeccable absolute ban + AEQI house rule).
- **Generic "rounded card with drop shadow" everywhere.** AEQI uses hairline borders + flat surfaces; shadows are reserved for popovers and modals.
- **Centered marketing-page layouts.** AEQI app surfaces are left-aligned, asymmetric where the content demands it.
- **Initial letter as accent on primitive nav** (a / e / q / i in the brand font). User has rejected this twice — use SVG icons on the rail.
- **Tabs with the brand font.** Use canonical tab bars, restrained.
- **"Awaiting" or "active" in jade.** Jade is success-only; use ink for attention.

### Design principles (binding for all impeccable work)

1. **Restraint over flourish.** One typographic gesture per section. Match neighbour weight. If you stack three editorial moments — Exo 2 hero + Zen Dots wordmark + a mono eyebrow + jade dot — you've already lost.
2. **Information density is the brand.** This is a working tool, not a marketing page. Resist the urge to add whitespace because it "feels professional." Linear-tight is the floor.
3. **Jade is sacred.** Reserved for success only. If you need an "attention" cue, use near-black ink at 100% or a hairline border. Never reach for a colour.
4. **Lowercase prose.** UI copy + headings + microcopy lean lowercase. ALL CAPS only for short mono labels (eyebrows, status pills).
5. **Click is navigate.** Rows are pointers; the work happens in detail views. No inline accordions, no inline edit, no embedded forms unless the row IS the form. Trust the session view.
6. **Mono for numbers.** Every count, ID, timestamp, currency value uses `var(--font-mono)` with `font-variant-numeric: tabular-nums`. This is non-negotiable for a precision tool.
7. **Hairlines beat boxes.** Group with `1px solid var(--border-faint)` dividers, not boxes-with-shadows. Boxes are reserved for genuinely separable surfaces (modals, popovers, the agent settings card).
